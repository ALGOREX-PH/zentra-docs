`ZentraVerifier` (Soroban) verifies proofs, enforces the state binding and
nullifier uniqueness, settles payments, and emits receipts.

## Methods

```rust
register_policy(agent, policy_commitment: BytesN<32>, recipient_root: BytesN<32>, epoch_seconds: u64)
authority_state(agent, policy: BytesN<32>) -> AuthorityState
revoke_policy(agent, policy: BytesN<32>) -> Result<(), Error>
verify_proof(proof: Proof, pub_signals: Vec<Fr>) -> Result<bool, Error>
authorize_action(
  agent, policy_commitment, proof_bytes: Bytes, recipient, amount: i128, asset,
  invoice_hash: BytesN<32>, nullifier: BytesN<32>,
  prev_epoch_id: u64, prev_spent: i128, prev_action_count: u64,
) -> Result<(), Error>
```

## `authorize_action` ‚Äî ordered state machine

Nothing past a failing step runs:

1. `agent.require_auth()`
2. assert `amount > 0` ‚Äî else `InvalidAmount`
3. load `PolicyRecord` ‚Äî `PolicyNotFound`; `PolicyRevoked` if revoked
4. compute `effective_prior` (epoch rollover)
5. assert `prev_*` == effective ‚Äî else `StateMismatch`
6. nullifier unused ‚Äî else `NullifierUsed`
7. `new_spent = prev_spent + amount`, `new_action_count = prev_action_count + 1` ‚Äî else `Overflow`
8. build 14 public inputs + Groth16 `verify` ‚Äî else `InvalidProof`
9. atomic writes: mark nullifier, write `AuthorityState`, extend TTLs
10. `token.transfer(agent ‚Üí recipient, amount)`
11. compute Poseidon `action_id`, emit `ActionReceipt`
12. `Ok(())`

## Events

`ActionReceipt` (topics `["receipt"]`):

```text
agent, policy, recipient, amount, asset, nullifier, epoch_id, new_action_count, action_id
```

## Storage

All persistent. `DAY_TTL = 17280` (~1 day at 5s ledgers), `RETENTION_TTL = 518400`
(~30 days). Nullifiers are persistent ‚Äî expiry would re-enable replay.

Built with `soroban-sdk` 26.1.0 (features `hazmat-address`, `hazmat-crypto`) and
`soroban-poseidon`; target `wasm32v1-none`.
