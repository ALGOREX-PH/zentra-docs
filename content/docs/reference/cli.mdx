The `zentra` binary (built on commander) wraps the SDK for terminal workflows.

## Commands

```bash
zentra init [--contract <id>] [--asset <asset>]

zentra policy create <name>
  --max-amount <n>
  --daily-limit <n>
  --allowlist <file>
  [--asset <asset>]
  [--epoch-seconds <n>]

zentra policy commit <file>

zentra prove <actionFile> --policy <file>

zentra submit <proofFile>
```

## `zentra.config.json`

```json
{
  "network": "testnet",
  "rpcUrl": "https://soroban-testnet.stellar.org",
  "networkPassphrase": "Test SDF Network ; September 2015",
  "contractId": "C…",
  "asset": "native",
  "circuit": { "wasmPath": "./payment_policy.wasm", "zkeyPath": "./payment_policy.zkey" }
}
```

## Scaffold & environment

`init` creates `policies/`, `actions/`, and `proofs/` alongside the config.

The agent's signing seed is read from the `ZENTRA_AGENT_SECRET` environment
variable (an `S…` testnet seed) and is never written to disk or logged.
